üíª Interactive Workflow Example

Workflows are defined in YAML files inside the .github/workflows directory. Click on the highlighted lines in this common Node.js CI workflow to see an explanation.

# .github/workflows/ci.yml

name: Node.js CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:

    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Run tests
        run: npm test

‚ú® Advanced & Productivity Features

Master these tools to handle more complex scenarios, improve security, and speed up your workflows.

Secrets Management

Store sensitive information like API keys or tokens as encrypted secrets in your repository settings. They can be used in your workflows via the ${{ secrets.YOUR_SECRET }} context, without ever exposing them in your code.

Deployment Environments

Define environments like 'staging' or 'production' with protection rules. This allows you to require manual approval from a specific team member before a deployment job can run, adding a crucial layer of security.

Dependency Caching

Use the actions/cache action to speed up your workflows. It stores dependencies (like `node_modules`) so they don't need to be downloaded from scratch on every run, which can significantly reduce job execution time.

Matrix Builds

Efficiently test your code across multiple versions of a language or on different operating systems. A matrix strategy creates a job for each configuration, allowing for comprehensive parallel testing with minimal code duplication.
